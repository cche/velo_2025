Pourquoi isoler les environnements?

Problèmes courants:

  • Conflits de versions entre projets
  • Dépendances incompatibles
  • Difficultés de reproduction
  • Pollution de l’environnement système

Solution:

  • Isolation des paquets et leurs versions
  • Indépendance par projet
  • reproductibilité par versions stables de tout l’environnement

Python Virtual Environments

Environnements virtuels Python

Concept de base:

  • Isolation des modules Python
  • Indépendance par projet
  • Gestion simple avec venv
# Créer un environnement
python3 -m venv mon_env

# Activer l'environnement
source mon_env/bin/activate

# Désactiver
deactivate

Gestion des dépendances

Fichier requirements.txt:

numpy==1.24.0
matplotlib==3.7.0
pytest==7.2.0

Installation:

pip install -r requirements.txt
pip freeze > requirements.txt
  • pip freeze permet de lister les modules et leurs versions installées dans l’environnement.
  • pip install -r permet de installer toutes les dépendances d’un fichier requirements.txt.

Conda (Miniconda)

Qu’est-ce que Conda?

Gestionnaire de paquets et d’environnements:

  • Gère Python ET les dépendances système
  • Supporte multiple langages
  • Résolution automatique des dépendances
  • Canaux de distribution (conda-forge)

Note

Miniconda: Version minimale de Anaconda

Installation de Miniconda

Sur Linux/WSL:

# Télécharger l'installeur
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh

# Installer
bash Miniconda3-latest-Linux-x86_64.sh

# Initialiser
conda init bash
source ~/.bashrc
  • Pour Mac avec Homebrew: brew install miniconda ou remplacer Linux par MacOS
  • conda init permet de configurer l’environnement shell actuel

Créer un environnement Conda

Commandes de base:

# Créer un environnement
conda create -n simulation python=3.11

# Activer
conda activate simulation

# Installer des packages
conda install numpy matplotlib pandas

# Désactiver
conda deactivate

Fichier environment.yml

Définir l’environnement:

name: chps
channels:
  - conda-forge
  - defaults
dependencies:
  - python=3.12
  - numpy
  - matplotlib
  - pip:
    - custom-package

Utilisation:

conda env create -f environment.yml
conda env export > environment.yml

Gestion des environnements

Commandes utiles:

# Lister les environnements
conda env list

# Supprimer un environnement
conda env remove -n simulation

# Mettre à jour conda
conda update conda

# Nettoyer le cache
conda clean --all

Docker

Introduction à Docker

Conteneurisation:

  • Isolation complète (OS, bibliothèques, code)
  • Portabilité garantie
  • Reproductibilité parfaite
  • Standard de l’industrie

Tip

Différence avec VM: Partage du noyau, plus léger

Concepts Docker

Éléments clés:

  • Image: Template immuable
  • Container: Instance d’une image en exécution
  • Dockerfile: Recette pour créer une image
  • Registry: Dépôt d’images (Docker Hub)

Installation Docker

Sur Linux:

# Installer Docker
sudo apt update
sudo apt install docker.io

# Ajouter l'utilisateur au groupe docker
sudo usermod -aG docker $USER

# Vérifier l'installation
docker --version
docker run hello-world

Installation Docker rootless

  • Besoin de:
    • newuidmap et newgidmap du paquet uidmap
    • dockerd-rootless-setuptool.sh
# Desactiver le service docker
sudo systemctl disable --now docker
sudo rm /var/run/docker.sock

# Installer Docker rootless
dockerd-rootless-setuptool.sh install

Instructions sur Docker

Dockerfile pour Python

Exemple simple:

FROM python:3.12-slim

WORKDIR /app

COPY . .
# COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

ENV PYTHONPATH=/app:$PYTHONPATH


CMD ["python", "mon_script.py"]

Commandes Docker essentielles

Build et Run:

# Construire une image
docker build -t velo-env .

# Lancer un container
docker run velo-env

# Mode interactif
docker run -it --rm --entrypoint /bin/bash velo-env

# Monter un volume
docker run -it --rm --entrypoint /bin/bash -v $(pwd):/app velo-env

# Detaché pour VSCode avec le repertoire monté
docker run -it -d --entrypoint /bin/bash -v $(pwd):/app velo-env

Gestion des containers

Commandes utiles:

# Lister les containers actifs
docker ps

# Lister tous les containers
docker ps -a

# Arrêter un container
docker stop <container_id>

# Supprimer un container
docker rm <container_id>

# Supprimer une image
docker rmi <image_id>

Singularity

Pourquoi Singularity?

Spécifique au HPC:

  • Pas besoin de privilèges root
  • Sécurité renforcée pour clusters
  • Compatible avec Docker
  • Accès direct au système de fichiers
  • Standard sur les clusters académiques

Singularity vs Docker

Différences clés:

Aspect Docker Singularity
Privilèges Root requis User-level
Sécurité Isolation forte Intégration système
Usage Dev, production HPC, recherche
Fichiers Volumes Bind mounts natifs

Installation Singularity

Sur Linux:

# Via package manager (Ubuntu)
sudo apt update
sudo apt install -y singularity-container

# Vérifier
singularity --version

Note

Note: Souvent pré-installé sur les clusters HPC

Créer une image Singularity

Fichier definition (.def):

Bootstrap: docker
From: python:3.12-slim

%post
    pip install numpy matplotlib

%files
    1_basic_single_sim/* /app/

%runscript
    python /app/run_single.py -step 1000 --init-mailly 10 --init-moulin 10 --seed 123 --out-csv output.csv --plot --p1 0.5 --p2 0.47

%labels
    Author cours.hpc@example.com
    Version v1.0

Build Singularity

Construction:

# Build depuis definition file (nécessite root/fakeroot)
sudo singularity build velo_env.sif velo_env.def

# Depuis Docker Hub
singularity build python.sif docker://python:3.12

# Depuis Docker local (root)
singularity build velo_env.sif docker-daemon://velo_env

# Depuis Docker local (rootless)
docker build -t velo-env .
docker save velo-env -o velo-env.tar
singularity build velo_env.sif docker-archive://velo_env.tar

Utiliser Singularity

Exécution:

# Exécuter le runscript
singularity run velo_env.sif

# Shell interactif
singularity shell velo_env.sif

# Exécuter une commande
singularity exec velo_env.sif python script.py

# Avec bind mount
singularity exec --bind /data:/mnt velo_env.sif python script.py

Singularity sur cluster HPC

Avec SLURM:

#!/bin/bash
#SBATCH --job-name=simulation
#SBATCH --ntasks=1
#SBATCH --time=01:00:00

module load singularity

singularity exec simulation.sif python /app/1_basic_single_sim/run_single.py -step 1000 --init-mailly 10 --init-moulin 10 --seed 123 --out-csv output.csv --plot

Singularity pour le cluster

Bootstrap: docker
From: python:3.12-slim

%post
    pip install numpy matplotlib pandas pytest

%files
    ./* /app/

%environment
    export PYTHONPATH=/app:$PYTHONPATH

%runscript
    if [ $# -eq 0 ]; then
        echo "Usage: singularity run velo_env.sif <script.py> [args...]"
        echo "Example: singularity run velo_env.sif /app/1_basic_single_sim/run_single.py --help"
        python --version
    else
        exec python "$@"
    fi
